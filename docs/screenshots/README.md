## 📷 Screenshot Guidelines

### Best Practices

1. **Clean State**
   - Use sample data that looks realistic
   - Avoid empty states unless demonstrating them
   - Ensure proper spelling and grammar

2. **Consistent Styling**
   - Use same window size across screenshots
   - Maintain consistent lighting/theme
   - Same font rendering settings

3. **High Quality**
   - Minimum 1920×1080 resolution
   - PNG format (lossless)
   - No compression artifacts

4. **Framing**
   - Show full window or relevant portion
   - Include window decorations if relevant
   - Crop unnecessary desktop space

### Capture Methods

#### Windows
```bash
# Built-in
Win + Shift + S  # Snip & Sketch

# Full window
Alt + PrintScreen  # Active window to clipboard

# Third-party
ShareX, Greenshot, Snagit
```

#### macOS
```bash
# Full screen
Cmd + Shift + 3

# Selected area
Cmd + Shift + 4

# Window
Cmd + Shift + 4, then Space
```

#### Linux
```bash
# GNOME
PrintScreen  # Full screen
Alt + PrintScreen  # Active window

# KDE
Meta + PrintScreen

# Third-party
Flameshot, Shutter, Spectacle
```

#### Flutter DevTools
```bash
# Capture from running app
flutter pub global activate devtools
flutter pub global run devtools

# Widget screenshots
# Use DevTools > Widget Inspector > Screenshot
```

## 🎨 Image Alt Text

When adding screenshots to README, always include alt text:

```markdown
![Schedule View - Rolling 7-day planner with color-coded task cards](schedule.png)
```

**Good Alt Text:**
- Describes what's shown
- Mentions key UI elements
- Accessible for screen readers

**Bad Alt Text:**
- "Screenshot 1"
- "App view"
- (empty)
